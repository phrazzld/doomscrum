First-run on-ramp lab doomscrum-942 · every empty state becomes an on-ramp · 8 structurally distinct options · self-contained mocks

Design question: how does a stranger get from doomscrum serve to a playing sample video (<60s) and a first consented dispatch (<2min) without ever leaving the app — and what does every dead-end (empty backlog, unrendered spec, no key, bad repo path, exhausted wallet) offer as its one obvious next action?

A — Feed-native on-ramp cards

No wizard, no new chrome. Every empty/error state renders as a feed card with the same visual grammar as a video card: a headline, one primary action, one secondary, and exact-path copy when file edits are genuinely required.

no specs found
backlog is empty
this repo has no specs in backlog.d/
sync a reporeload feed
or drop markdown specs into
~/code/myrepo/backlog.d/*.md and reload
unrendered spec, no key
no video yet
Fix the login flow
this spec has no video yet. fixture = free local placeholder, instant.
cook fixture (free)add FAL key → AI video
AI renders cost ~$0.35 each and send spec text to fal.ai — never required

tradeoffs

  • Preserves the feed-first identity — recovery lives where the failure happened.
  • Smallest diff: extends the existing render() empty branches.
  • Exact-command copy only where file edits are truly required (authoring specs).
  • No global "where am I in setup" view.

B — Persistent quest checklist

A "setup" chip beside the repo chip tracks four steps (repo → feed → key → first dispatch). Tapping it opens a checklist; each unfinished row is a button that performs or opens the fix.

checklist open
get cooking
✓ repo synced — doomscrum
✓ feed playing (3 samples)
○ FAL key — enable AI renders (~$0.35 ea) →
○ first dispatch — swipe right on a spec →
chip disappears once all four are done

tradeoffs

  • Gives strangers a persistent map; measurable funnel.
  • Familiar pattern (game quests, Linear onboarding).
  • New standing chrome for a one-time need; the key step nags the free path (019's oracle says the key is never required).
  • Doesn't fix dead-ends in place — you still need per-state recovery.

C — Story-mode splash (3 slides)

Expand the tap-in splash into three swipeable slides: (1) what this is, (2) the gesture grammar, (3) what right-swipe really does + costs. Feed starts after slide 3 or "skip".

→ dispatch a real agent → PR

↓ previous card

tradeoffs

  • Everything taught before the first card; consent framing pre-loaded.
  • Delays time-to-playing-video — works against the <60s oracle.
  • Wizard energy is off-brand for a brainrot feed; skippers learn nothing.
  • Doesn't address empty/error states at all — a second system still needed.

D — Coach layer on the first card

After the splash, the first card renders with a translucent one-time coach overlay naming the four gestures in place. Any interaction dismisses it forever (localStorage). Zero delay to the playing video underneath.

tradeoffs

  • Teaches the grammar at the moment of first use; video keeps playing under it.
  • One-time, self-dismissing, ~30 lines.
  • Alone it fixes none of the dead-ends — pairs with A or E.

E — Universal "fix it" recovery sheet

One shared bottom sheet for all failures: any dead-end (bad repo path, empty backlog, 429, no key) opens the same sheet listing the ranked next actions with the failure detail at top.

after a bad repo path
✗ /Users/x/nope has no backlog.d/
get unstuck
pick a recent repo
retry path
reload feed
a syncable repo is any directory with a backlog.d/ of markdown specs

tradeoffs

  • One consistent recovery affordance; easy to guarantee "no dead-end" invariant.
  • Generic sheet = worse copy than in-place recovery; failure context gets flattened.
  • Extra hop: failure → sheet → fix, vs fixing where you stand.

F — In-app doctor panel

Mirror the CLI's excellent doctor (6 named checks) as an in-app panel behind a chip; each failing check carries a fix button or the exact command.

doctor chip open
doctor
✓ repo · ✓ backlog (12 specs) · ✓ ffmpeg
✓ renders dir writable
✗ FAL key — add key
✗ gh auth — run: gh auth login
green = swipe away

tradeoffs

  • Reuses a proven mental model; complete diagnosis surface.
  • Names exact commands where terminal work is genuinely required (gh auth).
  • Diagnostic, not narrative — doesn't teach the first swipe or frame consequence.
  • Larger build (new API surface exposing preflight) for a stranger who mostly needs 2 buttons.

G — Consequence-framed consent (mission briefing)

The existing one-time consent overlay grows a consequence frame: THIS spec's title, THIS repo, and the three things that happen next, plus the undo window. The held-breath moment gets its copy.

first right swipe
dispatch a real agent?
spec: Fix the login flow
repo: doomscrum
1. a fresh git worktree is created
2. a coding agent implements this spec
3. a real pull request opens on this repo
undo: cancel on the card during the queue window (~5s)
Cook it — I get itcancel

tradeoffs

  • Exactly what the vision calls for: names what's about to happen, then delights.
  • Extends an existing shipped overlay — consent gate + undo preserved by construction.
  • Only covers the dispatch moment; pairs with A/D for the rest.

H — Ghost-hand auto-demo

On first run the top sample card animates itself: a ghost cursor drags it right to the threshold, the consent overlay peeks, then snaps back. Show, don't tell.

auto-demo mid-drag
▶ sample playing
Ship dark mode
→ ship it

tradeoffs

  • Most delightful teach; zero reading.
  • Risky near a real dispatch trigger; motion+state machinery is the biggest build here.
  • Fires once, helps nobody who blinked; still needs per-state recovery design.

Winner: A + D + G composite (in-place on-ramps · one-time coach · briefing consent)

B (checklist) and F (doctor panel) add standing chrome for a one-time need and nag the free path; C (wizard) works against the <60s oracle; E (universal sheet) flattens failure context; H (ghost demo) is the largest build for the least durable teaching. A, D, and G are complementary, not competing: A fixes every dead-end in place with one obvious action, D teaches the grammar at zero cost to time-to-video, and G gives the held-breath moment its consequence frame while preserving the shipped consent gate and undo window. The key sheet (from A's "add FAL key" action) is a single shared overlay quoting per-render price and the starter budget, never required for the free path. Smallest surface, feed-first identity intact.
