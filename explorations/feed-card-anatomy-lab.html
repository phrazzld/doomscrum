How every element is housed

  • caption — bottom of frame, flow layout (meme → plan line → degraded chip), pointer-events none. Nothing else may enter the zone in ANY state.
  • vibe — ledge row 1, below the frame. Always visible, never on video.
  • cook / retry / PR — ledge row 2 (chips appear per state).
  • status — chip inside top-right (no −12px overhang → clears the egress chip).
  • prio / review / superseded — chip column inside top-left.
  • agent log / queued-undo / failed+retry — HUD panel anchored to frame TOP, under the chips. Caption untouched while cooking.
  • meta (provider/model·sha) — demoted to the tap-spec overlay.
  • rail — stays vertical left; frame narrows to 86vw so dots clear it.
  • ticker — brand + N/M counter; subtitle dropped at phone width; zero truncation.

Trade

  • Occlusion-free caption guaranteed structurally — the interactive layer is not inside the frame at all.
  • All affordances stay one glance + one tap away; nothing hidden behind gesture discovery.
  • Smallest diff: same DOM ids, same gesture surface, e2e selectors survive.
  • Video frame is ~6vw narrower than today (86vw vs 92vw) to clear the rail.
  • Ledge costs ~80px of stage height below the frame (available at all three target viewports).

Winner: A — status ledge (borrowing G's position counter)

A is the only anatomy that makes caption occlusion structurally impossible in every dispatch state — the interactive layer is not inside the frame at all — without paying B/F/H's tax (affordances hidden behind discovery or gestures, agent states going dark) or C/G's tax (shrinking the brainrot). The vision's two non-negotiables decide it: "the brainrot communicates the spec" (caption zone is flow-only and pointer-inert, log/undo/retry live in a top HUD, meta demoted to the tap-spec overlay) and "consequences are legible" (vibe, cook-$, PR stay standing chrome, one glance and one tap away, and the cost confirmation graduates from window.confirm to a styled overlay consistent with consent/egress).

From G we steal exactly one idea: the ticker becomes brand + N/M position counter (no marquee corpse, no mid-sentence ellipsis) — but the rail keeps its vertical dots, cleared by narrowing the frame to 86vw. E's zone-swap and D's TikTok stack were the closest losers: E moves the one fixed thing in the scan rhythm, D starves sentence-length spec titles of width.

Implementation contract: same DOM ids (#card, .meme, consent ids) so gestures, keyboard parity, reduced-motion, and the browser e2e suite survive untouched; ledge is a sibling of the card inside a flex deck; status chips move inside the frame corners; logbox re-anchors top; cost overlay reuses the picker/overlay pattern quoting planned USD + daily budget.
